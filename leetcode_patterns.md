# 1. HashSet: проверка наличия

## Когда использовать

Используй `set`, если в задаче нужно:

- проверить, встречался ли элемент раньше;
- найти дубликат;
- быстро проверить наличие элемента;
- хранить только факт присутствия, без дополнительных данных.

## Типичные задачи

- `Contains Duplicate`
- проверка уникальности символов
- часть решения `Two Sum`

## Идея

Используй подсчёт частот, если в задаче есть:
Храним уже просмотренные элементы в множестве `seen`.

## Шаблон

```python
seen = set()

for x in nums:
    if x in seen:
        # элемент уже встречался
        ...

    seen.add(x)
```

## Инвариант

После обработки каждого элемента `seen` содержит все элементы, которые были до текущего.

## Типичные ошибки

### Ошибка 1: сначала добавить, потом проверить

Неправильно:

```python
seen = set()

for x in nums:
    seen.add(x)

    if x in seen:
        return True
```

Так условие всегда будет истинным для текущего элемента.

Правильно:

```python
seen = set()

for x in nums:
    if x in seen:
        return True

    seen.add(x)

return False
```

# 2. HashMap: найти пару / дополнение

## Когда использовать

Используй `dict`, если нужно:

- найти два числа, сумма которых равна `target`;
- для текущего элемента быстро найти подходящий предыдущий;
- хранить не только факт наличия, но и индекс / количество / другое значение.

## Типичные задачи

- `Two Sum`
- похожие задачи на поиск пары
- задачи вида “найти complement”

## Идея

Для текущего числа `x` ищем, встречалось ли раньше число `target - x`.

## Шаблон

```python
seen = {}

for i, x in enumerate(nums):
    need = target - x

    if need in seen:
        return [seen[need], i]

    seen[x] = i
```

## Инвариант

`seen` хранит элементы, которые стоят левее текущего индекса.

## Почему сначала проверка, потом добавление

Чтобы не использовать один и тот же элемент два раза.

Пример:

```text
nums = [3]
target = 6
```

Если сначала добавить `3`, а потом искать `3`, можно ошибочно взять один элемент дважды.

## Типичные ошибки

### Ошибка 1: перепутать ключ и значение

Обычно ключ — это число, а значение — индекс:

```python
seen[x] = i
```

Не наоборот.

### Ошибка 2: использовать один и тот же элемент дважды

Это возникает, если сначала добавлять элемент, а потом проверять complement.

# 3. Подсчёт частот

## Когда использовать

Используй подсчёт частот, если в задаче есть:

- анаграмма;
- уникальный символ;
- большинство элементов;
- “сколько раз встречается”;
- одинаковый набор символов;
- сравнение двух строк по составу.

## Типичные задачи

- `Valid Anagram`
- `First Unique Character in a String`
- `Majority Element`
- `Ransom Note`

## Шаблон через обычный dict

```python
cnt = {}

for x in items:
    cnt[x] = cnt.get(x, 0) + 1
```

## Шаблон через Counter

```python
from collections import Counter

cnt = Counter(items)
```

## Инвариант

`cnt[x]` показывает, сколько раз элемент `x` встретился в обработанной части.

## Пример: проверка анаграммы

```python
from collections import Counter

def is_anagram(s, t):
    return Counter(s) == Counter(t)
```

## Типичные ошибки

### Ошибка 1: проверять только множество символов

Неправильно:

```python
set(s) == set(t)
```

Контрпример:

```text
s = "aab"
t = "abb"
```

Множество символов одинаковое, но строки не анаграммы.

# 4. Группировка через словарь

## Когда использовать

Используй группировку через словарь, если нужно разложить элементы по группам:

- анаграммы;
- слова с одинаковым признаком;
- числа с одинаковым остатком;
- элементы с одинаковым ключом.

## Типичные задачи

- `Group Anagrams`
- группировка строк
- группировка чисел по остатку

## Идея

Создаём ключ, одинаковый для всех элементов одной группы.

Для анаграмм:

```text
eat -> aet
tea -> aet
ate -> aet
```

## Шаблон через обычный dict

```python
groups = {}

for item in items:
    key = get_key(item)

    if key not in groups:
        groups[key] = []

    groups[key].append(item)

return list(groups.values())
```

## Шаблон через defaultdict

```python
from collections import defaultdict

groups = defaultdict(list)

for item in items:
    key = get_key(item)
    groups[key].append(item)

return list(groups.values())
```

## Инвариант

`groups[key]` содержит все элементы, относящиеся к группе `key`.

## Типичные ошибки

### Ошибка 1: выбрать плохой ключ

Для анаграмм нельзя брать первый символ как ключ.

Контрпример:

```text
eat
ate
tea
```

Это анаграммы, но первые символы разные.

# 5. Two Pointers: slow / fast

## Когда использовать

Используй `slow / fast`, если нужно:

- удалить элементы из массива;
- переместить элементы;
- сжать массив;
- изменить массив `in-place`;
- вернуть новую длину массива.

## Типичные задачи

- `Move Zeroes`
- `Remove Element`
- `Remove Duplicates from Sorted Array`

## Идея

- `fast` просматривает все элементы;
- `slow` показывает позицию, куда нужно положить следующий подходящий элемент.

## Шаблон

```python
slow = 0

for fast in range(len(nums)):
    if nums[fast] подходит:
        nums[slow] = nums[fast]
        slow += 1
```

## Инвариант

До индекса `slow` лежат уже обработанные подходящие элементы.

```text
[хорошие элементы][мусор / ещё не обработано]
                  slow
```

## Типичные ошибки

### Ошибка 1: двигать slow всегда

Неправильно:

```python
for fast in range(len(nums)):
    if nums[fast] подходит:
        nums[slow] = nums[fast]

    slow += 1
```

`slow` надо двигать только тогда, когда мы реально положили хороший элемент.

# 6. Two Pointers: левый и правый указатель

## Когда использовать

Используй два указателя с двух концов, если:

- массив отсортирован;
- нужно найти пару;
- нужно проверить палиндром;
- нужно сравнивать начало и конец;
- можно постепенно сужать область поиска.

## Типичные задачи

- `Valid Palindrome`
- `Two Sum II`
- `Squares of a Sorted Array`
- `Container With Most Water` — позже

## Общий шаблон

```python
left = 0
right = len(arr) - 1

while left < right:
    if условие_выполнено:
        ...
    elif нужно_увеличить_значение:
        left += 1
    else:
        right -= 1
```

## Шаблон для суммы в отсортированном массиве

```python
left = 0
right = len(numbers) - 1

while left < right:
    current = numbers[left] + numbers[right]

    if current == target:
        return [left, right]
    elif current < target:
        left += 1
    else:
        right -= 1
```

## Инвариант

На каждом шаге мы выбрасываем часть вариантов, которые уже не могут быть ответом.

```text
left --->       <--- right
```

## Типичные ошибки

### Ошибка 1: двигать не тот указатель

Если сумма слишком маленькая, надо увеличить сумму:

```python
left += 1
```

Если сумма слишком большая, надо уменьшить сумму:

```python
right -= 1
```

# 7. Prefix Sum: сумма на отрезке

## Когда использовать

Используй префиксные суммы, если нужно:

- быстро считать сумму на отрезке;
- отвечать на много запросов суммы;
- работать с суммами подмассивов;
- оптимизировать повторяющийся подсчёт суммы.

## Типичные задачи

- задачи из блока 3 Яндекса;
- `Range Sum Query`;
- база для `Subarray Sum Equals K`.

## Идея

`prefix[i]` — сумма первых `i` элементов.

```text
prefix[i] = nums[0] + nums[1] + ... + nums[i - 1]
```

## Шаблон

```python
prefix = [0] * (len(nums) + 1)

for i in range(len(nums)):
    prefix[i + 1] = prefix[i] + nums[i]
```

Сумма на отрезке `[left, right]`:

```python
segment_sum = prefix[right + 1] - prefix[left]
```

## Инвариант

`prefix[i]` хранит сумму первых `i` элементов.

## Типичные ошибки

### Ошибка 1: путать индексы

Для суммы на отрезке `[left, right]` обычно нужно:

```python
prefix[right + 1] - prefix[left]
```

А не:

```python
prefix[right] - prefix[left]
```

# 8. Prefix Sum + HashMap

## Когда использовать

Используй этот паттерн, если задача спрашивает:

- сколько подмассивов имеют сумму `k`;
- есть ли подмассив с нужной суммой;
- нужно искать не саму сумму, а разницу между двумя префиксами.

## Типичные задачи

- `Subarray Sum Equals K`
- `Continuous Subarray Sum` — позже, сложнее

## Идея

Пусть текущая префиксная сумма равна `current`.

Нам нужно найти, была ли раньше такая префиксная сумма:

```text
current - k
```

Потому что:

```text
current - old_prefix = k
```

Значит участок между `old_prefix` и `current` даёт сумму `k`.

## Шаблон

```python
from collections import defaultdict

count = 0
current = 0
prefix_count = defaultdict(int)

prefix_count[0] = 1

for x in nums:
    current += x

    need = current - k
    count += prefix_count[need]

    prefix_count[current] += 1

return count
```

## Инвариант

`prefix_count[s]` — сколько раз раньше встречалась префиксная сумма `s`.

## Почему prefix_count[0] = 1

Это нужно для случая, когда подмассив начинается с индекса `0`.

Пример:

```text
nums = [3]
k = 3
```

Текущая сумма:

```text
current = 3
```

Нужная старая сумма:

```text
need = current - k = 0
```

Если заранее не записать, что сумма `0` была один раз до начала массива, ответ потеряется.

## Типичные ошибки

### Ошибка 1: обновлять prefix_count раньше проверки

Правильный порядок:

```python
count += prefix_count[need]
prefix_count[current] += 1
```

Если сделать наоборот, можно случайно учитывать текущую позицию как предыдущую.

# 9. Sliding Window: окно без повторов

## Когда использовать

Используй sliding window, если задача про:

- подстроку;
- подмассив;
- максимальную длину;
- минимальную длину;
- непрерывный участок;
- условие внутри окна.

## Типичные задачи

- `Longest Substring Without Repeating Characters`
- `Max Consecutive Ones`
- `Find All Anagrams in a String` — позже

## Идея

Есть окно `[left, right]`.

- `right` расширяет окно;
- `left` сжимает окно, когда условие нарушено.

## Шаблон для окна без повторов

```python
seen = set()
left = 0
best = 0

for right in range(len(s)):
    while s[right] in seen:
        seen.remove(s[left])
        left += 1

    seen.add(s[right])
    best = max(best, right - left + 1)

return best
```

## Инвариант

Внутри окна `[left, right]` нет повторяющихся символов.

## Типичные ошибки

### Ошибка 1: использовать if вместо while

Иногда нужно сдвинуть `left` несколько раз, пока окно снова не станет корректным.

Неправильно:

```python
if s[right] in seen:
    seen.remove(s[left])
    left += 1
```

Правильно:

```python
while s[right] in seen:
    seen.remove(s[left])
    left += 1
```

# 10. Binary Search

## Когда использовать

Используй бинарный поиск, если:

- массив отсортирован;
- нужно найти элемент;
- нужно найти первую / последнюю позицию;
- можно отбросить половину вариантов;
- есть монотонное условие.

## Типичные задачи

- `Binary Search`
- `Find First and Last Position of Element in Sorted Array`
- `Search Insert Position`
- `Find Minimum in Rotated Sorted Array` — позже

## Обычный шаблон

```python
left = 0
right = len(nums) - 1

while left <= right:
    mid = (left + right) // 2

    if nums[mid] == target:
        return mid
    elif nums[mid] < target:
        left = mid + 1
    else:
        right = mid - 1

return -1
```

## Инвариант

Ответ, если он есть, находится в диапазоне `[left, right]`.

## Типичные ошибки

### Ошибка 1: неправильное условие цикла

Для обычного поиска чаще используется:

```python
while left <= right:
```

Если использовать `left < right`, можно пропустить последний элемент.

### Ошибка 2: не сдвигать mid

Неправильно:

```python
left = mid
right = mid
```

Правильно:

```python
left = mid + 1
right = mid - 1
```

# 11. Stack: скобки и отмена последнего действия

## Когда использовать

Используй стек, если в задаче есть:

- скобки;
- вложенность;
- нужно проверить корректность порядка;
- нужно отменять последнее действие;
- нужен последний открытый элемент.

## Типичные задачи

- `Valid Parentheses`
- `Simplify Path`
- `Evaluate Reverse Polish Notation`

## Идея

Стек работает по принципу:

```text
last in — first out
```

Последний добавленный элемент извлекается первым.

## Шаблон для скобок

```python
stack = []
pairs = {
    ')': '(',
    ']': '[',
    '}': '{',
}

for ch in s:
    if ch in '([{':
        stack.append(ch)
    else:
        if not stack:
            return False

        if stack[-1] != pairs[ch]:
            return False

        stack.pop()

return len(stack) == 0
```

## Инвариант

В стеке лежат открывающие скобки, которые ещё не были закрыты.

## Типичные ошибки

### Ошибка 1: не проверить пустой стек

Неправильно:

```python
if stack[-1] != pairs[ch]:
    return False
```

Если стек пустой, будет ошибка.

Правильно:

```python
if not stack:
    return False
```

# 12. Linked List: разворот списка

## Когда использовать

Используй этот паттерн, если задача про:

- развернуть связный список;
- изменить связи между узлами;
- пройтись по списку без массива.

## Типичные задачи

- `Reverse Linked List`
- `Merge Two Sorted Lists`
- `Palindrome Linked List` — позже

## Идея разворота

Нужно последовательно менять направление ссылки `next`.

## Шаблон

```python
prev = None
current = head

while current:
    next_node = current.next
    current.next = prev

    prev = current
    current = next_node

return prev
```

## Инвариант

`prev` — уже развёрнутая часть списка.  
`current` — текущий узел, который ещё нужно обработать.

## Типичные ошибки

### Ошибка 1: потерять хвост списка

Нужно заранее сохранить:

```python
next_node = current.next
```

И только потом менять:

```python
current.next = prev
```

# 13. Tree DFS: рекурсия по дереву

## Когда использовать

Используй DFS, если задача про:

- глубину дерева;
- сравнение деревьев;
- путь от корня;
- проверку свойства в каждом узле;
- рекурсивную структуру.

## Типичные задачи

- `Same Tree`
- `Symmetric Tree`
- `Maximum Depth of Binary Tree`
- `Path Sum`

## Общий шаблон

```python
def dfs(node):
    if node is None:
        return базовое_значение

    left = dfs(node.left)
    right = dfs(node.right)

    return объединить(node.val, left, right)
```

## Инвариант

Функция `dfs(node)` решает задачу для поддерева с корнем `node`.

## Типичные ошибки

### Ошибка 1: забыть базовый случай

Без этого рекурсия уйдёт в `None` и сломается.

```python
if node is None:
    return ...
```

# 14. Tree BFS: обход по уровням

## Когда использовать

Используй BFS, если нужно:

- пройти дерево по уровням;
- найти минимальную глубину;
- работать с очередью;
- обрабатывать слой за слоем.

## Типичные задачи

- `Binary Tree Level Order Traversal`
- `Minimum Depth of Binary Tree`

## Шаблон

```python
from collections import deque

if root is None:
    return []

queue = deque([root])
result = []

while queue:
    level_size = len(queue)
    level = []

    for _ in range(level_size):
        node = queue.popleft()
        level.append(node.val)

        if node.left:
            queue.append(node.left)

        if node.right:
            queue.append(node.right)

    result.append(level)

return result
```

## Инвариант

В начале каждой итерации `while` в очереди лежат все узлы текущего уровня.

## Типичные ошибки

### Ошибка 1: не зафиксировать размер уровня

Если не сделать:

```python
level_size = len(queue)
```

то можно случайно смешать узлы текущего и следующего уровня.

# 15. Как выбирать паттерн

## Быстрая таблица

| Признаки задачи | Вероятный паттерн |
|---|---|
| есть дубликаты, наличие элемента | `set` |
| нужна пара с target | `dict`, complement |
| анаграммы, частоты | `Counter`, `dict` |
| группировка объектов | `dict` со списками |
| удалить/сжать массив in-place | `slow / fast` |
| отсортированный массив, пара, палиндром | `left / right pointers` |
| сумма на отрезке | `prefix sum` |
| количество подмассивов с суммой k | `prefix sum + hashmap` |
| подстрока/подмассив, longest/minimum | `sliding window` |
| отсортировано, отбросить половину | `binary search` |
| скобки, вложенность, последний элемент | `stack` |
| дерево, рекурсия | `tree DFS` |
| дерево по уровням | `tree BFS` |

# 17. Главный принцип

Паттерн — это не готовое решение.  
Паттерн — это способ быстро понять, в какую сторону думать.

Если задача не решилась, это не всегда значит, что ты “не понял алгоритмы”. Чаще это значит, что ты ещё не распознал паттерн или не удержал инвариант.
